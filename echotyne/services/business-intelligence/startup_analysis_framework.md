# Echotyne Startup Failure Analysis Framework

## Purpose
Extract patterns from failed startups to inform client advisory and identify opportunities.

## Methodology

### Step 1: Target Selection
Filter by:
- Industry alignment with client focus
- YC batch (recent = more relevant)
- Funding stage at failure
- Similar business model

### Step 2: Data Extraction
For each startup:
```
Company Name:
YC Batch:
Industry:
Funding Raised:
Years Active:
Official Reason for Failure:
Post-Mortem Insights:
What They Built:
What Succeeded Instead:
Key Mistakes:
Lessons for Clients:
```

### Step 3: Pattern Analysis
Look for:
- Common failure modes by industry
- Timing patterns (when do most fail?)
- Funding vs. runway correlations
- Market readiness indicators

### Step 4: Client Application
- Advisory: "Don't build X, build Y"
- Positioning: "We learned from Z's failure"
- Opportunity spotting: "This space is now open"

## Output Format

### Weekly Failure Digest
**Date:** [DATE]
**Analyzed:** [N] startups
**Focus:** [INDUSTRY]

**Key Findings:**
1. [Pattern 1]
2. [Pattern 2]
3. [Pattern 3]

**Client Recommendations:**
- [Actionable advice]

**Opportunities Spotted:**
- [Market gap]

## Tools
- Source: https://startups.rip
- Filter by batch, industry, status
- Cross-reference with Crunchbase for funding data
